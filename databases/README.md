# Databases From Scratch

Status: **Planned**

This track will study how software stores, retrieves, protects, and coordinates
persistent data.

## First-Principles Questions

- How is data represented on disk?
- Why do indexes make reads faster?
- What makes a transaction reliable?
- How do concurrent clients safely modify shared data?
- When should data be normalized or duplicated?

## Planned Outcomes

- Understand relational modeling and SQL
- Understand files, pages, indexes, and B-trees
- Work with transactions, isolation, and concurrency control
- Reason about query plans and performance
- Compare relational, document, key-value, and other database models
- Design and operate a database-backed application

Lessons and exercises will be added after the prerequisite tracks are complete.
