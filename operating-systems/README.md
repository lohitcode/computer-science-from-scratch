# Operating Systems From Scratch

Status: **Planned**

This track will study how an operating system manages hardware and provides the
abstractions used by programs.

## First-Principles Questions

- How can many programs share one CPU?
- What separates a process from a thread?
- How does virtual memory give each process its own address space?
- How do files and devices become usable through common interfaces?
- Why do concurrent programs deadlock or race?

## Planned Outcomes

- Understand processes, threads, scheduling, and system calls
- Understand virtual memory, paging, stacks, and heaps
- Work with files, descriptors, permissions, and signals
- Reason about synchronization, races, and deadlocks
- Understand the boundary between user space and the kernel
- Connect Go program behavior to operating-system mechanisms

Lessons and exercises will be added after the prerequisite tracks are complete.
