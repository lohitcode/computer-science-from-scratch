# Software Security From Scratch

Status: **Planned**

This track will study how to protect software, users, and data by understanding
threats and designing secure boundaries.

## First-Principles Questions

- What are we protecting, and from whom?
- How do authentication and authorization differ?
- Why is untrusted input dangerous?
- What security properties does cryptography provide?
- How can systems remain secure when individual components fail?

## Planned Outcomes

- Build threat models and identify trust boundaries
- Understand authentication, authorization, sessions, and credentials
- Prevent common input, web, API, and dependency vulnerabilities
- Understand hashing, encryption, signatures, certificates, and TLS
- Manage secrets and permissions safely
- Add security checks throughout development and deployment

Lessons and exercises will be added after the prerequisite tracks are complete.
