# System Design From Scratch

Status: **Planned**

This track will study how individual components are combined into reliable,
scalable, and maintainable software systems.

## First-Principles Questions

- How do requirements determine architecture?
- What changes when one machine is no longer enough?
- Where should state live?
- How should a system behave when a dependency fails?
- Which trade-offs matter more than fashionable technology choices?

## Planned Outcomes

- Translate requirements into capacity and reliability needs
- Design APIs, services, storage, caching, and messaging boundaries
- Reason about replication, partitioning, consistency, and availability
- Handle overload, retries, timeouts, idempotency, and partial failure
- Compare monoliths, modular systems, and distributed services
- Communicate designs with diagrams and explicit trade-offs

Lessons and exercises will be added after the prerequisite tracks are complete.
